Описание класса шаблонизатора

Шаблонизатор использует несколько констант для управления своей работой. ВСЕ они необязательные и проинициализированы значениями поумолчанию, если не задано другого.

Прежде всего они используются для удобства и управления (корректирования) автоматическими функциями.

Ниже перечислены имена и значения поумолчанию, а также указано на что и как влияют.

Описание языка разметки для данного шаблонизатора представлено здесь.

Сам класс описан здесь.

На данный момент определены следующие константы:
TEMPLATE_DEBUG = false| true Если true, после всего результата, выводится список всех ошибок и варнингов в порядке их появления в шаблоне. Также внизу страницы выводится время генерации шаблона в Сек.
TEMPLATE_GENTIMECALC = false| true Если true, вычисляется и (и если TEMPLATE_DEBUG=true в конце выводится) время парсинга шаблона
TEMPLATE_TEMPLATES_DIR =
= none | <директория откуда взят и сам файл класса шаблонизатора> 		Директория шаблонов, используемая поумолчанию. Имя директории БЕЗ завершающего слеша.

Используется в методе find_path

Настоятельно рекомендую задавать всегда!

TEMPLATE_MODULES_SUBDIR = 'modules/' поддиректория, где искать модули шаблонизатора
TEMPLATE_DEFAULT_SCHEME = 'default.scheme' Схема, используемая поумолчанию. Имя файла.
TEMPLATE_DEFAULT_SCHEME_DIR =
= <директория откуда взят и сам шаблон> 		Директория схем, используемая поумолчанию. Имя директории БЕЗ завершающего слеша.
TEMPLATE_MESSAGE = 'message.tmpl' Файл шаблона, который будет считаться фаблоном поумолчанию для метода message
