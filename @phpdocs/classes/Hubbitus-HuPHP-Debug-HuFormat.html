$MODS

HuFormat.php : 63

For each present Mod we must have method with name "mod_[mod]" where [mod] is letter of mode.

public static array<string|int, mixed> $MODS = array( 'A' => 1, //ALL. Exclusive, all other modifiers not processed. Each process as HuFormat. 's' => 2, //Setting 'a' => 4, //Array 'n' => 8, //Non_empty_str 'p' => 16, //sPrintf. {@link http://php.net/sprintf} 'e' => 32, //Evaluate. Evaluated only ->_name !!! 'E' => 64, //Evaluate full! Evaluate all as full result. 'v' => 128, //Value, 'I' => 256, //Iterate ->_value (or ->_realValue) and each format as ->_format 'k' => 512, )

Additionally, because PHP function (methods too) name are case insensitive, for upper-case letter modifiers must used double letters. For example: Mod 'e' => mod_e Mod 'E' => mod_EE (same as mod_ee)

cloning()

HuClass.php : 39

This is just wrapper for system construction 'clone'. Objects in PHP implicitly returned by reference (see http://www.php.net/manual/en/language.references.spot.php#59820), so, to use it without modification of main object I should clone it, but it is break one line construction chain.

public static cloning(mixed $obj) : mixed

F.e. I want only count such items: $dellin->getNotMatchedCities()->filter(create_function('$v', 'return is_null($v->region);'))->count() it implicitly MODIFY $dellin object! Off course if it is not return clone of object itself, what is not a common case. So, we want do something similar: (clone $dellin->getNotMatchedCities())->filter(create_function('$v', 'return is_null($v->region);'))->count() But PHP does not allow such construction and fire there parsing error. For this case the method intended. In our example it would: HuClass::clone($dellin->getNotMatchedCities())->filter(create_function('$v', 'return is_null($v->region);'))->count()

Some developer notes:

  • Unfortunately we can't name it as just clone even in class because it is reserved word.
  • I use clone in method because argument itself again implicitly passed as reference, so it is required.

setFormat()

HuFormat.php : 168

Parse incoming format-array and set appropriate properties.

public & setFormat(array<string|int, mixed>|string $format) : mixed

Accepts 3 forms of format (in examples ':::' is ::parseModsName()):

  1. All in one Array: [0] - Mods:::Name; [1],[2],[3]...[n] - data. array( 'sn:::bold_text', //Mods:::Name '', '', 'default text' //Data )

  2. Associative array (hash): Key - Mods:::Name, value - Array of [0] - Mods, [1], [2]...[n] - data. array( //Name 'bold_text' => array( //Mods empty, Name '', '', 'default text' //Data ) )

  3. Just simply string like 'text to add'. Leaved as is.
